Nifty PM Theme Guidelines and Documetation

Gutenberg Editor

The new Nifty theme uses the Gutenberg editor, which is now Wordpress's default editor. It has a different interface but coincides nicely with the development of our theme. It's use of blocks supports our system of shortcode usage (described in this document) and allows us several benefits not previously available to us. The Classic Editor plugin that has made Wordpress look the same for ages, which has been used on Nifty's theme up to this point, will be abandoning support in 2021/2022, so we will no longer rely on it going forward.

We use the block editor extensively for housing our shortcode sections.

Shortcodes

Our new themes rely almost entirely off of shortcode functionality. Shortcodes are one way that Wordpress allows us to make the same design or use the same blocks over and over. They output the same HTML and CSS while allowing for content changes. There are two types of shortcodes: 1) standalone and 2) wrapping.

Standalone Shortcodes

Standalone shortcodes look like this:

[shortcode_name]

Standalone shortcodes are helpful for outputting static content or design that will rarely ever change from page to page.

Wrapping Shortcodes

Wrapping shortcodes will often look like this:

[SHORTCODE_NAME]

text text text

[/SHORTCODE_NAME]

The difference in a wrapping shortcode from a standalone shortcode is that the wrapping shortcode will output the content between its tags. We use wrapping shortcodes to give PMs control over the content they will see on the front end. In our current practices, we will capitalize all wrapping shortcodes to show the distinction (though this has not always been the case in older iterations of our theme).

Attributes

Attributes are another way of giving design and content control to PMs. Attributes will look like this:

  • Standalone shortcode:
    [shortcode_name title="something" color="blue" button_text="Call Today"]

  • Wrapping Shortcodes:
    [SHORTCODE_NAME heading="Don't Wait" bg_img="89" text_color="green"]
    Content content content
    [/SHORTCODE_NAME]

Shortcode attributes are pre-defined by the developer. They correspond to what you see on the front end and give PMs the control over different parts of a block template. All attributes must have and equals sign (=) followed by double or single quote marks that open and close (' ') or (" ").

Any punctuation or HTML using quotes or double quotes should be encoded as to not mess up the shortcode rendering. This can be done by replacing them with the following characters:

  • Single quote – &apos;
  • Double quote – &quot;

Editing Content

To edit content on the Nifty-created themes, it may require editing PHP pages (PHP is a backend coding language. Even if you don't completely understand PHP or HTML, you should be able to make these edits fairly simply without breaking anything. The following sections explain how to do this efficiently.

Theme Editor No-Touch Files

Most of the development for a site , which includes content,  is created in the theme editor under the "Appearance" menu sidebar of the Wordpress backend. This includes directories (folders) and PHP files.

The following are directories and files which are off-limits to PMs or unauthorized personnel:

  • Stylesheet (style.css)
  • Theme Functions (functions.php)
  • 404 Template (404.php)
  • Theme Footer (footer.php)
  • Theme Header (header.php)
  • includes (folder and contents)
  • Main Index Template (index.php)
  • Single Page (page.php)
  • Search Results (search.php)
  • Single Post (post.php)
  • template-parts/ada-components/ (folder and contents)
  • template-parts/content/ (folder and contents)
  • template-parts/blog-styles.php
  • Any AJAX files
  • Any AJAX pages

No explanation for these files will be provided in this documentation but are copiously explained in our extensive developer documentation for anyone who takes and interest. Manipulating these files without developer experience can cause different elements of the site to break or can cause the entire site to crash. These are used to control the overall site and are important, but don't control individual content or block styles.

Included among these is now the header.php and footer.php. The functionality of these files is included elsewhere (explained in the next section).

Them Editor Do-Touch Files

The following backend, theme editor directores and PHP pages are safe for your access. Each will be explained below:

  • Pm-safe (folder and contents)
    • after-body-scripts.php
    • footer-scripts.php
    • header-scripts.php
  • template-parts/blocks
  • template-parts/pages (folder and contents)
  • template-parts/section (folder and contents)

PM-Safe

The pm-safe directory contains three files: 1) after-body-scripts.php, 2) footer-scripts.php, and 3) header-scripts.php.

header-scripts.php: If you have any extra code that goes in the header of a site, including things such as analytics, meta tags, or Facebook tracking (among other codes and scripts), they will be placed here instead of the header.php (as it was previously). This PHP file is being pulled directly into the header so the header.php file doesn't need to be edited directly.

footer-scripts.php: If you have any extra code that goes in the footer of a site, such as Callrail or other scripts, those will be placed here instead of the footer.php (as it was previously). This PHP file is being pulled directly into the footer so the footer.php file doesn't need to be edited directly.

after-body-scripts.php: There are a few scripts that require loading directly after the <body> tag of a site. Any scripts requiring this, including some Facebook tracking scripts and Google analytics integrations, will go in this file and will be included in the correct spot across the site.

template-parts/block (folder and contents)

This directory and its contents contains content for blocks used across the site. If any content is needed to be edited within a block, those blocks may be accessed and changed here.

template-parts/pages (folder and contents)

Individual pages often have their own design templates and blocks which are not used on other parts of the site. Those blocks of content will be found here and can be edited as needed.

template-parts/section (folder and contents)

Most shortcodes will be created in this directory. They are usually divided into three main sections, but there may be more depending on the project:

  1. Homepage: These are shortcodes that are almost exclusively created for the homepage.
  2. Internal: These are shortcodes that are almost exclusively created for internal pages.
  3. Custom: These are shortcodes that don't quite fit the "blocks" category, a "page" category, the "homepage" category, or the "internal" category but are still needed for creating the site.

How to Know Where to Edit Content

All content on our new theme will follow a simple process so you can find and edit content easily. Content is created in three parts: 1) shortcode name, 2) the shortcode's php page, and 3) the HTML class. These are all named the same so you can easily find where each piece of content is located.

For instance, if you see a banner image on the homepage and see that the overall section is labeled with the class "home-banner", you will see that the shortcode is called "[home_banner]". The php on the backend will be called "home-baner.php". If you needed to make an adjustment to the home banner, you would simply navigate your way back to the PHP page where you could make that update.

There is a simple process to follow whenever you want to find where content is to edit it.

  1. Use developer tools on the front-end to see what the section class is called.
  2. Find the content in the Gutenberg block editor for that section (it will be titled the same thing as the class on the front-end).
  3. Change the attribute or adjust the wrapping shortcode content if it is there.
  4. If the content isn't there, find the corresponding shortcode in the backend files.

For your convenience, if you want to edit content –

  • On the homepage:
    • corresponding shortcode name will be [home_{name}]
    • corresponding PHP page will be home-{name}.php.

  • On an internal page:
    • corresponding shortcode name will be [internal_{name}]
    • corresponding PHP page will be internal-{name}.php.

  • On a main page (like contact or practice areas):
    • corresponding shortcode name will be [page_{name}]
    • corresponding PHP page will be page-{name}.php.

  • Controlled by a custom shortcode/section:
    • corresponding shortcode name will be [custom_{name}]
    • corresponding PHP page will be custom-{name}.php.

  • Controlled by a block shortcode/section:
    • corresponding shortcode name will be [block_{name}]
    • corresponding PHP page will be block-{name}.php.

HTML Sitemap

As part of our new theme, there is an out-of-the-box shortcode we created to include an HTML sitemap called [nifty_sitemap]. This will include all pages and posts on the backend of a site, including nofollow and noindex pages. To exclude any page from the HTML sitemap, add the attribute exclude="" and include any page ids you do not want to include in the sitemap, separated by a comma. Your shortcode will likely look something like this:

[nifty_sitemap exclude="977, 960, 882, 979, 1335"]

Adding Inline Images and Iframes

If you are adding content to a blog or page and want to include an iframe video/embed or image, you will want to change the "src" to "data-url" instead. This assists in our lazy loading processes and will ensure higher page speed scores for those pages with embeds

AJAX Pages

A newly developed site will likely have a few noindex, nofollow pages that are titled "AJAX: {Name of the Page}". These should be left untouched. They are being used to control different aspects of the site and to facilitate lazy loading. They shouldn't appear in the XML or HTML sitemap of any site and should be excluded in the nifty_sitemap shortcode (though the developers will usually do this for you before launch).
